# IMPACT_ANALYSIS.md — <NomDuProjet>

> À remplir **avant** toute nouvelle fonctionnalité ou modification. C'est le réflexe de sécurité du volet Code : on cartographie l'impact avant de toucher au code.
> Couplé à la recette d'ajout de feature du kit de rails (`docs/kit-de-rails.md`) : la recette est le rail générique, cette analyse est l'instance précise du changement.

## Format d'une analyse

```
### IA-XXX — Changement envisagé
- **Objet** : ce qu'on veut faire (lien F-XXX si applicable).
- **Fichiers concernés** : liste précise des fichiers à modifier ou créer.
- **Fichiers à NE PAS modifier** : zones sensibles à préserver (voir ARCHITECTURE.md).
- **Impacts attendus** : effets sur le comportement, les données, les autres modules.
- **Risques** : ce qui peut casser, régressions possibles.
- **Tests à lancer** : quels tests valident ce changement (voir TEST_PLAN.md).
- **Validation humaine requise ?** : oui / non, et pourquoi.
```
