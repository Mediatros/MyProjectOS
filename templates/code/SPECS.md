# SPECS.md — <NomDuProjet>

> Les fonctionnalités attendues, décrites côté besoin avant côté technique.
> Une spec claire précède l'implémentation. Les détails longs peuvent être déportés dans `05_specs/`.

## Format d'une fonctionnalité

```
### F-XXX — Nom de la fonctionnalité
- **Besoin** : à quoi ça sert, pour qui.
- **Comportement attendu** : ce que le système doit faire.
- **Critères d'acceptation** : conditions vérifiables qui prouvent que c'est fait.
- **Hors périmètre** : ce que cette feature ne fait pas.
- **Lien** : SPECS détaillée (05_specs/), TEST_PLAN, IMPACT_ANALYSIS.
```
